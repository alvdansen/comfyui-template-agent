    - iter_all_nodes yields nodes from top-level and subgraphs
    - check_core_node_preference flags custom nodes with warning severity
    - check_no_set_get_nodes flags SetNode/GetNode/Set/Get with error severity
    - check_note_color_black flags Note/NoteNode with non-dark bgcolor as error
    - check_unique_subgraph_names flags duplicate subgraph names as error
    - check_subgraph_rules flags Preview*/Save* nodes inside subgraphs as error
    - detect_api_nodes matches known nodes + pattern on class_type_pattern
    - run_validation: format gate -> run registry -> filter by mode -> assemble report
    - Lenient mode: only severity=error findings
    - ignore list: skips rules by ID
  </behavior>
  <action>
    1. Create `src/validator/api_nodes.py`:
       - `load_api_node_data() -> dict`: Load `data/api_nodes.json`, cache in module-level variable
       - `detect_api_nodes(workflow: dict) -> list[Finding]`: For each node in workflow (use iter_all_nodes from rules.py):
         - Check class_type against known_nodes lists (exact match)
         - If no exact match, check if class_type contains any class_type_pattern (substring)
         - For each match, create Finding with rule_id="api_node_auth", severity=Severity.WARNING, message naming the provider, suggestion="Requires Comfy.org account authentication ({auth_type})"
         - Also store node_id and node_type on the Finding

    2. Create `src/validator/rules.py`:
       - `_load_core_nodes() -> frozenset[str]`: Load data/core_nodes.json once, return frozenset of node names. Cache in module variable.
       - `_load_guidelines() -> dict[str, dict]`: Load data/guidelines.json, return dict keyed by rule ID. Cache in module variable.
       - `iter_all_nodes(workflow: dict)`: Generator yielding every node dict from `workflow["nodes"]` and `workflow.get("definitions", {}).get("subgraphs", [])` each subgraph's `nodes` list.
       - Rule functions (all signature: `(workflow: dict) -> list[Finding]`):
         - `check_core_node_preference`: Use iter_all_nodes. For each node, if node["type"] not in core_nodes frozenset, create Finding(rule_id="core_node_preference", severity=WARNING, message=f"Custom node '{type}' (node {id}) - not a core node", suggestion="Consider if a core ComfyUI node can achieve the same result")
         - `check_no_set_get_nodes`: Match type against `{"SetNode", "GetNode", "Set", "Get"}`. severity=ERROR.
         - `check_note_color_black`: Match type in `{"Note", "NoteNode"}`. Check `bgcolor` and `color` fields. If present and not dark (not starting with "#0" and not "#000000"), create ERROR finding. If color fields absent, create INFO finding suggesting to set black background.
         - `check_unique_subgraph_names`: Read `workflow.get("definitions", {}).get("subgraphs", [])`. Collect names. If any name appears more than once, ERROR finding.
         - `check_subgraph_rules`: For each subgraph, iterate its nodes. If any node type starts with "Preview" or "Save", ERROR finding.
         - `check_api_node_color_yellow`: For nodes matched by detect_api_nodes, check if color is set. INFO reminder if no color set.
         - `check_group_color_default`: Check `workflow.get("groups", [])`. For each group with a non-None `color` field, WARNING finding.
         - `check_cloud_compatible`: INFO reminder that workflow must be tested on Comfy Cloud.
         - `check_thumbnail_specs`: INFO reminder about thumbnail requirements.
         - `check_api_badge_position`: INFO reminder about top-left corner.
         - `check_simplicity_readability`: INFO reminder about clarity and universality.
         - `check_naming_conventions`: INFO reminder about naming patterns.
       - `RULE_REGISTRY: dict[str, callable]` mapping each rule ID from guidelines.json to its function.
       - `SEVERITY_MAP = {"required": Severity.ERROR, "recommended": Severity.WARNING}` for guidelines severity -> Finding severity baseline.

    3. Create `src/validator/engine.py`:
       - `run_validation(workflow: dict, mode: str = "strict", ignore: list[str] | None = None) -> ValidationReport`:
         1. Call `detect_format(workflow)` from `src.shared.format_detector`. If not "workflow", return immediately with a report that has `passed=False`, one error Finding saying "Expected workflow format, got {format}. Convert API format to workflow format before validating."
         2. Load guidelines from rules._load_guidelines()
         3. Build ignore set from ignore parameter
         4. For each rule in RULE_REGISTRY:
            - If rule_id in ignore set, increment skipped counter, continue
            - Call the rule function with workflow
            - Build RuleResult: passed = len(findings) == 0, findings = findings
         5. If mode == "lenient", filter each RuleResult's findings to severity=ERROR only. Re-evaluate passed based on filtered findings.
         6. Also call detect_api_nodes and add as a RuleResult with rule_id="api_node_auth"
         7. Compute overall: passed = all results passed, score = f"{passed_count}/{checked_count} rules passed"
         8. Build summary: flatten all findings, sort by severity (error first), take top 10
         9. Return ValidationReport
  </action>
