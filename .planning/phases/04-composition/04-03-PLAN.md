Create `.claude/skills/comfy-compose/SKILL.md` with a Claude Code skill that guides template composition.

The skill should document:

**Description**: Compose valid ComfyUI workflow JSON through guided building or template scaffolding. Uses MCP node specs for type-safe connections and auto-populated widget defaults.

**Prerequisites**: ComfyUI MCP server must be connected (comfyui-cloud or comfyui-mcp). Test by calling `search_nodes` — if it fails, instruct user to start ComfyUI + MCP server.

**Two Starting Paths**:
1. **Goal-based** (common patterns): User describes what they want ("I want a Flux img2img workflow"). Claude searches templates, scaffolds the closest match, then modifies.
2. **Node-first** (custom builds): User names specific nodes. Claude composes from scratch using WorkflowGraph.

**Composition Workflow**:
1. Determine starting path (goal-based or node-first).
2. If goal-based: search templates with `search_templates` MCP tool or `python -m src.templates.search`, scaffold closest match.
3. If node-first: create empty WorkflowGraph, add nodes one by one.
4. For each node: call MCP `search_nodes` to get full spec. Use `parse_node_spec` to convert. Feed to graph's NodeSpecCache.
5. Ask user about key parameters: model, resolution, steps, CFG. Auto-default everything else from spec defaults.
6. Connect nodes with type checking. Flag any custom node usage ("Note: X is a custom node — template guidelines prefer core nodes, but it's the best option for this task").
7. Run auto_layout.
8. Save with `save_workflow` which runs validation.
9. Show validation report. Fix any issues.
10. Offer to submit to Comfy Cloud via `submit_workflow` MCP tool for testing.
11. Auto-generate title and description based on nodes used and workflow purpose.

**Flags**:
- `--scaffold <template_name>`: Start from existing template
- `--file <path>`: Start from local workflow file
- `--output <path>`: Output path (default: current directory)
- `--no-validate`: Skip validation (not recommended)

**Usage Examples**:
```
"compose a Flux img2img workflow"
"scaffold from wan-t2v-basic and swap the model loader"
"compose a workflow using KSampler, VAEDecode, and SaveImage from scratch"
"load workflow.json and add an upscale step"
```

**Key Behaviors** (from CONTEXT.md locked decisions):
- Suggest best available nodes regardless of core/custom — flag custom for awareness
- Auto-default most parameters, ask about model, resolution, steps
- Auto-generate titles and descriptions from workflow content
- Support basic subgraphs (wrapping nodes into named subgraph)
- Cloud-first option: user can submit to Comfy Cloud at any point, not just at the end
- All scaffold operations: swap nodes, change params, add/remove nodes, rewire

**CLI Shortcut**: `python -m src.composer.compose --scaffold <name> --output workflow.json`

Follow the exact structure and formatting of `.claude/skills/comfy-validate/SKILL.md`.
